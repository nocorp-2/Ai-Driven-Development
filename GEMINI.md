# Описание проекта

Данный проект представляет собой базу знаний (документацию), организованную в виде хранилища **Obsidian** (о чем свидетельствует наличие папки `.obsidian/`). Основная тематика проекта — **AI-разработка (AI Dev)**, рабочие процессы с использованием ИИ-агентов (в частности Claude Code и MCP-серверов), оркестрация (Swarms, DAG-фреймворки, Memory Bank) и применение ИИ для точной реализации UI/UX дизайна (Pixel-Perfect).

## Структура директорий

*   **`Чат DEKSDEN/`**: Содержит концептуальные и архитектурные материалы по AI-разработке, системному дизайну, оркестрации пайплайнов, а также управлению затратами и трендам индустрии.
*   **`SaveAs/`**: Библиотека полезных ресурсов, включающая списки LLM-навыков (MCP серверов для Claude), готовые промпты для различных задач (стратегия, контент, поиск работы) и ссылки на внешние каталоги/инструменты.
*   **`.obsidian/`**: Конфигурационные файлы приложения Obsidian (настройки внешнего вида, плагинов и воркспейса).

## Ключевые файлы

*   **`AI_Dev_Workflow_Recommendations.md`**: Сборник лучших практик, подходов и антипаттернов при работе с ИИ-агентами. Описывает концепции Memory Bank, DAG-оркестрации и важность предварительного планирования перед написанием кода.
*   **`Pixel_Perfect_Design_Implementation.md`**: Подробное руководство по верстке интерфейсов с использованием ИИ, акцентирующее внимание на создании строгих контрактов (`design-spec.md`) и CSS-переменных до начала кодирования.
*   **`Чат DEKSDEN/AI_разработка.md`**: Документ о сдвиге парадигмы в разработке — переходе от простого чата с ИИ к полноценным агентным пайплайнам, Spec-Driven Development и подходам Domain-Driven Design (DDD).
*   **`SaveAs/llms_and_skills/claude_skills.md`**: Обширная база данных MCP-серверов и навыков для расширения возможностей Claude Code (от UI/UX и архитектуры до управления памятью и автоматизации браузера).

## Использование

Этот репозиторий не содержит исполняемого кода, скриптов сборки или тестов (это не-кодовый проект). Он предназначен для:
1.  Открытия и редактирования с помощью приложения **Obsidian**.
2.  Изучения передовых практик разработки с помощью искусственного интеллекта.
3.  Поиска конкретных инструментов, фреймворков и промптов для интеграции в рабочие процессы.

## Конвенции и рекомендации для агента

*   При получении вопросов, связанных с архитектурой ИИ-агентов, лучшими практиками разработки или инструментами, в первую очередь ищите ответы в локальных Markdown-файлах этого проекта.
*   Особое внимание стоит уделять философии проекта: "Планирование важнее кодирования", использование строгих спецификаций и контрактов.
*   Сохраняйте разметку Markdown и структуру ссылок при редактировании или добавлении новых материалов, чтобы не нарушать целостность графа Obsidian.
