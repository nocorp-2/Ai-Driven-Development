# AI-Driven Development Knowledge Base

Добро пожаловать в репозиторий базы знаний, посвященной передовым практикам AI-разработки (AI Dev). Этот проект представляет собой организованное хранилище **Obsidian**, содержащее концепции, архитектурные паттерны, рабочие процессы и инструменты для эффективного использования ИИ-агентов (таких как Claude Code), MCP-серверов и фреймворков оркестрации.

## 🎯 Ключевые особенности

- **Spec-Driven Development:** Подходы к разработке, основанные на спецификациях и строгих контрактах.
- **Оркестрация агентов:** Паттерны использования Swarms, DAG-фреймворков и суб-агентов для решения сложных задач.
- **Управление контекстом:** Практики применения Memory Bank для предотвращения потери фокуса и раздувания контекста.
- **UI/UX и Pixel-Perfect:** Методологии применения ИИ для точной верстки и интеграции с дизайн-системами.
- **Библиотека ресурсов:** Списки полезных LLM-навыков, MCP-серверов и готовых промптов.

## 🚀 Начало работы (Local Setup)

Так как этот проект является базой знаний Obsidian, для комфортной работы с ним вам не потребуются компиляторы или среды выполнения кода.

### Открытие проекта в Obsidian

1. Установите [Obsidian](https://obsidian.md/download).
2. Запустите программу и выберите опцию **"Open folder as vault"** (Открыть папку как хранилище).
3. Укажите путь к склонированной папке данного проекта.
4. Obsidian автоматически подтянет все настройки из папки `.obsidian/` (включая тему и плагины).

## 🧠 Ключевые концепции базы знаний

### 1. Планирование важнее кодирования
Детальное планирование, создание жестких контрактов (`design-spec.md`) и спецификаций до написания кода позволяет сэкономить токены и избежать эффекта "спагетти-кода".

### 2. Управление состоянием (Memory Bank)
Сохранение контекста проекта между сессиями агента и формирование "Lessons Learned", чтобы избегать повторения одних и тех же ошибок.

### 3. Направленная оркестрация (DAG)
Вместо сложных циклических процессов используется подход направленных ациклических графов (DAG). Работа разбивается на строго последовательные задачи.
