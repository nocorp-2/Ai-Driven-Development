# AI-Driven Development Knowledge Base

Добро пожаловать в репозиторий базы знаний, посвященной передовым практикам AI-разработки (AI Dev). Этот проект представляет собой организованное хранилище **Obsidian**, содержащее концепции, архитектурные паттерны, рабочие процессы и инструменты для эффективного использования ИИ-агентов (таких как Claude Code), MCP-серверов и фреймворков оркестрации.

## 🎯 Ключевые особенности

- **Spec-Driven Development:** Подходы к разработке, основанные на спецификациях и строгих контрактах.
- **Оркестрация агентов:** Паттерны использования Swarms, DAG-фреймворков и суб-агентов для решения сложных задач.
- **Управление контекстом:** Практики применения Memory Bank для предотвращения потери фокуса и раздувания контекста.
- **UI/UX и Pixel-Perfect:** Методологии применения ИИ для точной верстки и интеграции с дизайн-системами.
- **Богатая библиотека ресурсов:** Списки полезных LLM-навыков, MCP-серверов и готовых промптов.

## 🚀 Начало работы (Local Setup)

Так как этот проект является базой знаний Obsidian, для комфортной работы с ним вам не потребуются компиляторы или среды выполнения кода.

### 1. Клонирование репозитория

```bash
git clone https://gitlab.zoloto585.ru/ecom/ai-driven-development.git
cd ai-driven-development
```

### 2. Установка Obsidian

Если у вас еще не установлен Obsidian, скачайте его с официального сайта: [obsidian.md](https://obsidian.md/download).

### 3. Открытие проекта в Obsidian

1. Запустите Obsidian.
2. Выберите опцию **"Open folder as vault"** (Открыть папку как хранилище).
3. Укажите путь к склонированной папке `ai-driven-development`.
4. Obsidian автоматически подтянет все настройки из папки `.obsidian/` (включая плагины, такие как `terminal`, и темы оформления).

## 🧠 Ключевые концепции базы знаний

### 1. Планирование важнее кодирования
Основная философия проекта заключается в том, что детальное планирование, создание жестких контрактов (`design-spec.md`) и спецификаций до написания кода позволяет сэкономить токены и избежать эффекта "спагетти-кода", который часто возникает при неконтролируемой работе ИИ.

### 2. Управление состоянием (Memory Bank)
Использование систем вроде Memory Bank позволяет сохранять контекст проекта между сессиями агента, формировать "Lessons Learned" и избегать повторения одних и тех же ошибок.

### 3. Направленная оркестрация (DAG)
Вместо сложных циклических процессов рекомендуется использовать направленные ациклические графы (DAG) для оркестрации агентов. Эпик разбивается на строго последовательные задачи и подзадачи.

## 🤖 Рекомендации для ИИ-агентов

Если вы — ИИ-агент (например, Claude Code или Gemini CLI), работающий в этом репозитории, соблюдайте следующие правила:

1. **Контекст прежде всего:** При ответах на вопросы о лучших практиках AI-разработки, сначала изучите локальные Markdown-файлы.
2. **Сохранение ссылок:** При добавлении или редактировании файлов сохраняйте совместимость ссылок в формате Obsidian (`[[Название статьи]]` или стандартный Markdown).
3. **Русский язык:** Все материалы базы знаний и ответы должны быть на русском языке.
