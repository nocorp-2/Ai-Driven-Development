# Системный дизайн: Агент = Модель + Упряжка

В сообществе укоренилось понимание, что ИИ-агент — это сложная программная система, а не просто "умный чат-бот".

## Метафора Гурджиева
Пользователи сравнивают архитектуру агента с классической метафорой из философии:
1. **Лошадь (LLM)** — двигатель и вычислительная мощь. Тянет всю систему, но без контроля ускачет в сторону (галлюцинации).
2. **Повозка (Harness / Упряжка)** — обвязка, тулы, MCP (Model Context Protocol), память. Сама никуда не поедет.
3. **Кучер (Agentic Workflow)** — процесс управления и контроля графа. Держит вожжи (system prompts / steer files) и направляет лошадь.
4. **Пассажир (Human-in-the-loop)** — человек-оператор, ставящий высокоуровневую цель.

## Реверс-инжиниринг Claude Code
Упоминается исследование (статья *Dive into Claude Code: The Design Space of Today’s and Future AI Agent Systems*), где авторы разобрали исходники агента Anthropic. 
- **Главный вывод:** 98.4% кодовой базы — это инфраструктура (безопасность, парсинг, тулы), и лишь менее 2% — логика работы с самой LLM.
- **Инсайт:** Надежность автономных систем достигается детерминированной инженерной обвязкой, а не промпт-инжинирингом. На первый план выходит *System Design*.

### Ключевые архитектурные находки
Помимо базового соотношения кода, исследование выявило критические паттерны, которые делают Claude Code надежным production-инструментом:

1. **OS-подобная архитектура (Ядро и Демоны):**
   LLM выступает в роли "ядра" операционной системы (kernel), а вся обвязка (безопасность, память, инструменты) функционирует как демоны пользовательского пространства. Базовый паттерн — асинхронный цикл генерации ReAct (Reason + Act), работающий через единый `queryLoop`.

2. **Пятиуровневый конвейер сжатия контекста (Compaction Pipeline):**
   Для борьбы с переполнением контекстного окна применяется каскадная система очистки:
   - *Budget reduction:* Использование ссылочных указателей вместо полного текста.
   - *Snip:* Обрезка нерелевантной истории.
   - *Microcompact:* Кэш-ориентированное сжатие локальных блоков.
   - *Context collapse:* Семантическое объединение блоков.
   - *Auto-compact:* Резюмирование истории самой моделью.

3. **Шлюз безопасности (Deny-first Permission Gate):**
   Строгая проверка разрешений перед выполнением любого действия. Модель никогда не взаимодействует с файловой системой напрямую; запросы проходят через детерминированную функцию оценки (декларативные правила и ML-классификатор намерений).

4. **Механизмы расширения (по влиянию на токены):**
   Архитектура предусматривает 4 уровня расширяемости:
   - **Hooks (Хуки):** Триггеры и скрипты, выполняемые до/после действий.
   - **Skills (Навыки):** Кастомные промпты и цепочки (например, `.faf` или `.md`).
   - **Plugins (Плагины):** Внутренние расширения логики CLI.
   - **MCP (Model Context Protocol):** Стандартизированные внешние серверы для интеграции.
