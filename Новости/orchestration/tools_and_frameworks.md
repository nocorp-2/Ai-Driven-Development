# Инструменты и Фреймворки (Harness)

Сообщество пришло к выводу, что чистая LLM — это лишь ядро (лошадь), а реальную ценность приносит "Упряжка" (Harness/Framework). 

## Популярные концепции и тулы

- **Superpowers:** Управляет процессом разработки. Заставляет агента пройти фазу мозгового штурма (brainstorming), планирование, вести работу через TDD (Test-Driven Development) и отдельно проверять результат (верификация). Это слой инженерной дисциплины. Фреймворк «агентских навыков» и полноценная методология разработки ПО, адаптированная для ИИ-агентов. Включает Subagent-Driven Development с двухэтапной проверкой качества и системную отладку (4-фазный процесс поиска первопричин).
- **Beads:** Отвечает за состояние проекта (state). Распределенный графовый трекер задач для ИИ-агентов, работающий на базе Dolt (SQL-база данных с контролем версий). Хранит задачи в git (с поддержкой веток и слияний), сохраняет контекст между сессиями, учитывает зависимости и блокировки (позволяя ИИ видеть общую картину). Есть фичи Memory Decay (семантическое сжатие старых закрытых задач) и Stealth Mode (использование локально без фиксации в основном репозитории). Это полноценный task-tracking и «апгрейд памяти» для кодинг-агентов.
- **Template Bridge:** Связующее звено. Раскладывает идеи на эпики и подзадачи, встраивает их в workflow.
- **cmux:** Терминал на базе Ghostty с вертикальными табами, браузером и уведомлениями от агентов. Работает как универсальный UI для CLI-агентов (Claude Code, Codex, OpenCode). Своего рода "Cursor для CLI".

## Глобальные конфигурации
- **`CLAUDE.md` и `AGENTS.md`:** Файлы-инструкции для агентов. Зачастую используются симлинки (symlinks), чтобы один файл (например, `AGENTS.md`) был главным источником правды для всех утилит.
- Инструмент вроде `npx skills` позволяет устанавливать скиллы (кастомные промпты и тулы) прямо в репозиторий через симлинки.
- **Claude Code Templates:** CLI-инструмент и хаб готовых конфигураций для Anthropic Claude Code. Служит центральным хранилищем для расширения возможностей ИИ-агента через специализированные роли (агенты), кастомные слэш-команды, MCP интеграции и инструменты мониторинга (вкл. мобильный интерфейс для просмотра чатов).
