# 4 Фазы рабочего процесса BMAD

Процесс разработки в BMAD Method — это линейный пайплайн (сборочная линия), который проходит через четыре обязательные стадии. Переход к следующей стадии возможен только после завершения и документирования предыдущей.

## 1. Analysis (Анализ)
*   **Цель:** Глубокое понимание задачи.
*   **Участники:** Человек + ИИ-Аналитик.
*   **Что происходит:** На этом этапе вы общаетесь с ИИ, рассказываете свою идею. ИИ должен задавать вам вопросы, критиковать идею, находить слабые места. Это стадия свободного диалога.
*   **Выходной артефакт:** Документ `project-brief.md`.

## 2. Planning (Планирование)
*   **Цель:** Формализация требований.
*   **Участники:** ИИ-PM.
*   **Что происходит:** На основе брифа создается детальное описание продукта. Что именно мы строим? Кто пользователи? Как они взаимодействуют с системой?
*   **Выходной артефакт:** Документ `prd.md` (Product Requirements Document).

## 3. Solutioning (Проектирование решения)
*   **Цель:** Создание технического дизайна.
*   **Участники:** ИИ-Архитектор + ИИ-Scrum Master.
*   **Что происходит:** ИИ получает `prd.md` и формирует структуру папок, схемы БД, контракты API. Затем этот большой план разбивается на шаги (эпики и стори).
*   **Выходные артефакты:** 
    *   Документ `architecture.md`.
    *   Папка `_bmad/steps/` с отдельными файлами задач (например, `step-01-database-schema.md`, `step-02-auth-api.md`).

## 4. Implementation (Реализация)
*   **Цель:** Написание и тестирование кода.
*   **Участники:** ИИ-Разработчик + ИИ-QA.
*   **Что происходит:** 
    1. ИИ-Разработчик берет **ровно один файл** (например, `step-01-database-schema.md`), читает `architecture.md` для контекста и пишет код.
    2. Вы или ИИ-QA проверяете результат.
    3. Если всё верно, шаг помечается как выполненный, контекст чата сбрасывается (начинается новый чат).
    4. Берется следующий файл (`step-02`).
*   **Выходные артефакты:** Рабочий код в вашем проекте.

## Главное правило пайплайна
**Код пишется только на фазе 4.** 
Любые изменения в бизнес-логике или новых функциях, которые приходят вам в голову во время 4-й фазы, должны возвращаться на фазу 1 или 2. Нельзя просто попросить ИИ в чате: «А добавь-ка сюда еще интеграцию со Stripe». Нужно добавить это в PRD, обновить Архитектуру, создать новый Step-файл и только потом писать код.
