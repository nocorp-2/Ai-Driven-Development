# Введение в BMAD Method

**BMAD** (Breakthrough Method for Agile AI-Driven Development, также расшифровывается как *Build More Architect Dreams*) — это современный фреймворк для разработки программного обеспечения, созданный Брайаном Мэдисоном (Brian Madison). Он переносит принципы классического Agile-менеджмента в эпоху ИИ-агентов.

## Проблема: Vibe Coding (Интуитивный кодинг)

Большинство разработчиков начинают работу с ИИ-ассистентами (Claude, ChatGPT, Cursor) с подхода, который называют **«Vibe Coding»**:
1. Открывается пустой чат.
2. Разработчик пишет абстрактный промпт (например, «Сделай мне приложение для управления задачами»).
3. ИИ начинает писать код.
4. Разработчик просит внести правки, затем еще правки, затем еще...

**Почему это перестает работать на масштабе?**
- **Потеря контекста (Context Loss):** Чем длиннее чат, тем больше ИИ «забывает» изначальные требования.
- **Галлюцинации:** Пытаясь удержать в памяти архитектуру, дизайн и логику одновременно, модель начинает придумывать несуществующие переменные или нарушать уже написанный код.
- **Сжигание токенов (Token Burn):** Каждое новое сообщение заставляет ИИ перечитывать всю огромную историю чата.

## Решение: Спецификационно-ориентированный подход (Spec-Driven Development)

BMAD предлагает совершенно иную парадигму. Вместо того чтобы сразу просить ИИ написать код, вы используете ИИ для создания детального **бумажного проекта** (спецификаций, диаграмм, PRD). 

ИИ рассматривается не как умный автодополнитель кода, а как полноценная **команда разработки**, где каждый виртуальный «сотрудник» выполняет только свою узкую задачу.

### Ключевые преимущества BMAD:
1. **Предсказуемость результата:** Вы точно знаете, что будет закодировано, еще до написания первой строчки кода.
2. **Экономия до 80% токенов:** Каждая задача решается в новом, чистом окне контекста. ИИ скармливаются только те файлы, которые нужны ему прямо сейчас.
3. **Масштабируемость:** Метод позволяет работать с крупными легаси-базами (Brownfield), где необходимо строго соблюдать существующие паттерны.
