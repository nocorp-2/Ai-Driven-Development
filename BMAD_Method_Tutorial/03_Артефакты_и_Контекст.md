# Артефакты и Контекстная инженерия (Artifact-Driven State)

Самая революционная (и в то же время простая) идея BMAD заключается в том, **где хранится состояние (State) вашего проекта**.

В интуитивном кодинге (Vibe Coding) состояние хранится в истории чата с ИИ. В BMAD **состояние хранится в Markdown-файлах на вашем диске.**

## Контекстная инженерия (Context Engineering)

Современные LLM (Claude 3 Opus/Sonnet, GPT-4) имеют огромные окна контекста (до 200k токенов). Но загрузка всего проекта в каждый запрос — это:
1. Очень дорого (или быстро сжигает лимиты в Cursor/Claude).
2. Запутывает модель (чем больше мусора в контексте, тем хуже ИИ фокусируется на главном).

**Контекстная инженерия BMAD** — это принцип: «Предоставляй агенту только ту информацию, которая ему абсолютно необходима для выполнения текущей задачи».

### Как это работает на практике?
При выполнении задачи `step-04-login-component.md` агенту-Разработчику не нужно знать схему базы данных биллинга или бизнес-требования к рассылке email. Ему нужно скормить только:
1. Общий файл `architecture.md` (чтобы понимать глобальный стек и стилистику).
2. Файл текущей задачи `step-04-login-component.md` (где описаны UI, стейт-менеджмент и критерии приемки).
3. Существующие компоненты кнопок/инпутов (UI-кит), если они уже есть.

Всё остальное игнорируется. Это обеспечивает лазерный фокус ИИ на конкретной задаче.

## Ключевые артефакты BMAD

Обычно BMAD инициализирует специальную скрытую папку (например, `_bmad/` или `.bmad/`) в корне вашего проекта, где хранятся эти артефакты:

1.  **`project-brief.md`** — Видение и цели (high-level).
2.  **`prd.md`** — Пользовательские истории, функциональные/нефункциональные требования.
3.  **`architecture.md`** — Стек, компоненты, контракты данных, соглашения по коду (линтеры, правила именования).
4.  **`steps/` (Директория с шагами)** — Файлы вида `step-01.md`, `step-02.md`. 
    *   Каждый такой файл содержит:
        *   Название задачи.
        *   Критерии приемки (Acceptance Criteria).
        *   Список файлов, которые нужно создать или изменить.
        *   Статус (TODO, IN PROGRESS, DONE).

Когда вы используете ИИ-IDE (например, Cursor), вы можете просто упомянуть файл `@step-01.md` и дать команду: *"Реализуй эту задачу, сверяясь с @architecture.md"*. ИИ получает идеальный, стерильный контекст и выдает блестящий результат.
