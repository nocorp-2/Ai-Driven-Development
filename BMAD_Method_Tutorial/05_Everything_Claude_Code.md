# Экосистема "Everything Claude Code"

Репозиторий **[everything-claude-code](https://github.com/affaan-m/everything-claude-code)** (созданный победителем хакатона Anthropic) представляет собой мощный набор инструментов, навыков и настроек для ИИ-агентов (в частности, для Claude Code CLI). 

Эта экосистема идеально дополняет философию BMAD Method, предоставляя готовые технические решения для реализации ролевой модели и контекстной инженерии. 

## Как это связано с BMAD?

Если BMAD — это **методология** (как правильно организовать процесс), то `everything-claude-code` — это **готовый набор инструментов**, который автоматизирует этот процесс для вашего ИИ-агента. Обе системы сходятся в главных идеях: планирование важнее кодинга, а делегирование узким специалистам лучше универсального чата.

## Ключевые возможности и концепции

### 1. Специализированные агенты (48+ ролей)
В репозитории собраны готовые системные промпты и конфигурации для десятков специализированных ИИ-ролей. Это напрямую реализует концепцию Personas из BMAD:
*   **Планировщик (Planner)** — аналог Аналитика/PM.
*   **Архитектор (Architect)** — проектирование систем.
*   **Ревьюер (Reviewer)** — аналог QA-инженера.
*   Узкие специалисты (например, эксперт по безопасности или сборке).

### 2. Навыки (Skills) и Готовые воркфлоу (180+)
Вместо того чтобы учить ИИ с нуля, как проводить аудит безопасности или применять TDD (Test-Driven Development), вы подключаете готовые «навыки».
*   **Research-first (Сначала исследование):** Навык, который заставляет агента глубоко исследовать кодовую базу (читать файлы, искать зависимости), прежде чем предлагать какие-либо изменения или писать код.
*   **Spec-driven (Разработка по спецификации):** Автоматическая генерация детальных чертежей (blueprints) и планов до начала кодирования (например, команда `/plan`).

### 3. Автоматизация жизненного цикла (Hooks)
Хуки (Hooks) позволяют автоматизировать рутину вокруг работы ИИ. 
*   **Сохранение контекста:** ИИ может автоматически сохранять свои знания и текущий статус в конце сессии и загружать их в начале новой. Это решает проблему потери контекста на больших проектах.
*   **Проверка кода:** Настроены хуки, которые заставляют агента самостоятельно проверять код линтерами и прогонять тесты сразу после редактирования файла.

### 4. Непрерывное обучение (Continuous Learning)
Система умеет извлекать паттерны из ваших успешных сессий с ИИ. Если вы вместе с агентом решили сложную проблему, этот процесс можно превратить в новый многоразовый «навык» (Skill) или «инстинкт» для будущих сессий.

## Главные уроки из Everything Claude Code

При интеграции этой системы в ваш BMAD-воркфлоу, помните о следующих правилах:

1. **Контроль контекстного окна:** Избыток подключенных инструментов (MCP-серверов) или слишком много одновременно загруженных навыков могут переполнить контекст модели. Отключайте то, что не нужно для текущего этапа (например, отключайте MCP-сервер для работы с Git, если сейчас идет этап формирования бизнес-требований).
2. **Безопасность (Security-first):** При делегировании задач ИИ-агентам возрастают риски. В репозитории уделяется внимание сканированию уязвимостей и контролю доступа агентов к вашей системе (концепция AgentShield).
3. **Строгие контракты:** Как и в BMAD, успех агента зависит от качества постановки задачи. Использование команды `/plan` перед написанием кода — залог того, что ИИ не начнет галлюцинировать.

## Резюме

**Everything Claude Code** — это, пожалуй, лучший на сегодняшний день (2025-2026) открытый инструментарий для того, чтобы превратить ваш терминал с ИИ в полноценную команду разработчиков, работающую по принципам Spec-Driven Development. Вы можете использовать его пресеты для быстрого старта фаз Analysis, Solutioning и Implementation в рамках BMAD.
