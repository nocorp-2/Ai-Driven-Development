# Шаг 3: Настройка системы управления контекстом и задачами (State Management)

**Статус:** 🔴 Обязательно

## Зачем это нужно?
Модели LLM (в т.ч. Claude) со временем забывают первоначальную задачу (эффект "снежного кома контекста") или начинают "галлюцинировать", если история чата слишком большая. Управление состоянием (task-tracking) решает эту проблему, заставляя агента вести дневник в репозитории.

## Memory Bank

**Memory Bank** — это директория (например, `memory-bank/`), в которой агент хранит актуальное состояние проекта. 

### Базовая структура файлов:
*   `projectbrief.md` — Главная цель проекта и требования.
*   `activeContext.md` — Текущая задача, над чем сейчас работаем.
*   `productContext.md` — Как продукт должен работать с точки зрения пользователя.
*   `systemPatterns.md` — Архитектурные решения и паттерны.
*   `progress.md` — Что уже сделано и что осталось.

### Инструкция по запуску:
1. Создайте в корне проекта файл `memory-bank-rules.md` (инструкцию для агента по ведению этих файлов).
2. Напишите агенту в терминале (в сессии Claude Code): 
   > "Инициализируй Memory Bank для этого проекта. Опиши нашу текущую цель в `projectbrief.md` и создай `activeContext.md`".
3. **Главное правило:** В конце каждой сессии агент должен обновлять `activeContext.md` и `progress.md`, чтобы в следующей сессии (при перезапуске `claude`) он мог прочитать эти файлы и сразу "вспомнить", на чем остановился.

## Альтернатива: Beads
Фреймворк, использующий похожий подход: хранение стейта в графе Markdown файлов. Подходит для сложных проектов с большим количеством взаимосвязей. Выбор между Memory Bank и Beads зависит от личных предпочтений, но суть одна — **хранить контекст в файлах, а не в памяти чата**.
