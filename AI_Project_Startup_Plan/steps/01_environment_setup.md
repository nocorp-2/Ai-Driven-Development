# Шаг 1: Установка и базовая настройка среды (Claude Code, MCP & Templates)

**Статус:** 🔴 Обязательно

## Зачем это нужно?
Отказ от классического чата в браузере в пользу работы напрямую в терминале с доступом к файловой системе и инструментам (MCP). Это дает агенту возможность читать, анализировать и менять код автономно. Использование шаблонизаторов экономит время на конфигурацию.

## Установка и запуск

1. **Глобальная установка Claude Code:**
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```
   *Запуск в директории проекта:*
   ```bash
   cd my-project
   claude
   ```

2. **Использование шаблонизатора (claude-code-templates):**
   Вместо ручного создания конфигов, перейдите в корень вашего проекта и запустите:
   ```bash
   cd my-project
   npx claude-code-templates
   ```
   *Пример использования:* 
   Появится интерактивное меню. Выберите стек (например, `frontend-developer`), отметьте нужные MCP-серверы (например, `github`, `postgresql`). Утилита автоматически сгенерирует конфигурационные файлы и, при необходимости, подключит мониторинг сессий (Dashboard на aitmpl.com).

## Конфигурация среды (Примеры файлов)

Важнейшие файлы, которые должны лежать в корне проекта:

**`settings.json`** (Настройка таймаутов и лимитов):
```json
{
  "theme": "dark",
  "verbose": true,
  "mcp": {
    "timeoutMs": 30000
  }
}
```

**`mcp-servers.json`** (Декларативное подключение интеграций):
```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "YOUR_TOKEN_HERE" 
      }
    }
  }
}
```
*(Никогда не коммитьте `mcp-servers.json` с реальными ключами в git! Используйте `.gitignore` или загружайте токены через переменные окружения ОС).*

## Оптимизация токенов
**Проблема:** Каждый MCP-сервер передает агенту свои описания инструментов, что тратит контекст и деньги.
**Решение:** Ограничивайте количество MCP (не более 10) и инструментов (не более 80). Отключайте неиспользуемые серверы в `mcp-servers.json` перед стартом сессии.
