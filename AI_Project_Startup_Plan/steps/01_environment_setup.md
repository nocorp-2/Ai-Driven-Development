# Шаг 1: Установка и базовая настройка среды (Claude Code, MCP & Templates)

**Статус:** 🔴 Обязательно

## Зачем это нужно?
Отказ от классического чата в браузере в пользу работы напрямую в терминале с доступом к файловой системе и инструментам (MCP). Это дает агенту возможность читать, анализировать и менять код автономно. Использование шаблонизаторов экономит время на конфигурацию.

## Установка и запуск

1. **Глобальная установка Claude Code:**
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```
   *Запуск в директории проекта:*
   ```bash
   cd my-project
   claude
   ```

2. **Использование шаблонизатора (claude-code-templates):**
   Вместо ручного создания конфигов, перейдите в корень вашего проекта и запустите:
   ```bash
   cd my-project
   npx claude-code-templates
   ```
   *Пример использования:* 
   Появится интерактивное меню. Выберите стек (например, `frontend-developer`), отметьте нужные MCP-серверы (например, `github`, `postgresql`). Утилита автоматически сгенерирует конфигурационные файлы и, при необходимости, подключит мониторинг сессий (Dashboard на aitmpl.com).

## Конфигурация среды (Примеры файлов)

Важнейшие файлы, которые должны лежать в корне проекта:

**`.claude/settings.json`** (Настройка поведения, таймаутов и минимизации прерываний):
Для уменьшения количества отвлекающих запросов на подтверждение действий, настройте явные списки разрешенных и запрещенных команд:
```json
{
  "theme": "dark",
  "verbose": true,
  "mcp": {
    "timeoutMs": 30000
  },
  "allowedTools": ["bash", "read_file", "replace"],
  "askTools": ["run_shell_command"],
  "deniedTools": []
}
```

**`.claude/commands/`** (Кастомные slash-команды):
Создавайте пользовательские команды для частых сценариев. Например, команду `/council` для вызова "Совета ботов", или `/review` для запуска проверок.

## Режимы работы (Операционное управление)
*   **`acceptEdits`**: Включите этот режим для непрерывного написания кода без постоянных запросов на подтверждение каждого изменения.
*   **`/effort xhigh`**: Используйте эту встроенную команду перед постановкой сложных архитектурных задач, чтобы выделить агенту больше токенов и времени на размышление.

**`mcp-servers.json`** (Декларативное подключение интеграций):
```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "YOUR_TOKEN_HERE" 
      }
    }
  }
}
```
*(Никогда не коммитьте `mcp-servers.json` с реальными ключами в git! Используйте `.gitignore` или загружайте токены через переменные окружения ОС).*

## Рекомендуемые (Must-have) MCP-серверы
Согласно передовым практикам, для качественной работы настоятельно рекомендуется подключить следующие серверы:
1. **Context7**: Для получения актуальной документации по библиотекам и фреймворкам. Предотвращает галлюцинации и использование устаревших API.
2. **Sequential Thinking**: Помогает агенту выстраивать пошаговые рассуждения и проверять гипотезы, особенно при сложной отладке.
3. **Playwright**: Дает агенту "глаза" — возможность открывать браузер, проверять верстку, ошибки в консоли браузера и работу фронтенда.

## Оптимизация токенов
**Проблема:** Каждый MCP-сервер передает агенту свои описания инструментов, что тратит контекст и деньги.
**Решение:** Ограничивайте количество MCP (не более 10) и инструментов (не более 80). Отключайте неиспользуемые серверы в `mcp-servers.json` перед стартом сессии.
