# Шаг 8: Многоуровневая верификация, Code Review и Continuous Learning

**Статус:** 🔴 Обязательно

## Зачем это нужно?
Нельзя доверять сгенерированному коду вслепую. После завершения задачи требуется обязательная верификация (чтобы не сломать проект) и рефлексия (чтобы агент постоянно обучался).

## Code Review и Верификация
Проверка созданного кода перед слиянием. Для этого привлекается роль Reviewer (или используется другая модель).

*Пример команды:*
> "Переключись на роль 'Строгий Senior Reviewer'. Проверь последние изменения в `src/api.ts` на соответствие изначальным контрактам (из `CLAUDE.md`) и спецификациям (из `auth-spec.md`). Ищи уязвимости и проблемы с оптимизацией. Выведи список замечаний."

## Continuous Learning (Непрерывное обучение)
Автоматическое извлечение паттернов и успешных решений из сессий в многоразовые "инстинкты" агента. Вместо того чтобы каждый раз заново объяснять агенту, как настраивать CI/CD, вы формализуете этот опыт.

### Инструмент: Skill Creator
Доступен через пайплайны `everything-claude-code`. Позволяет сгенерировать новые навыки на основе диалогов и git-истории.

*Пример запуска в Claude Code:*
> "/skill-creator Проанализируй, как мы только что настроили CI/CD пайплайн для GitHub Actions, и создай навык `github-actions-setup` в `SKILL.md`, чтобы в будущем ты мог делать это автономно по одной команде."

**Итог:** Созданный навык сохраняется в проекте в виде отдельного файла. В следующей сессии агент автоматически подтянет этот контекст, когда вы попросите его: "Настрой GitHub Actions для нового микросервиса", без необходимости промптить его с нуля.
