# Шаг 4: Привлечение специализированных ролей (Аналитик / Архитектор)

**Статус:** 🟡 Опционально (Рекомендуется для средних и крупных проектов)

## Зачем это нужно?
Один универсальный ИИ-агент (генералист) хуже справляется со сложной архитектурой, чем специализированная "персона". Проекты вроде `everything-claude-code` предоставляют готовые пресеты ролей (Архитектор, Планировщик, Ревьюер).

## Установка everything-claude-code

1. **Глобальная установка пайплайнов и плагинов:**
   ```bash
   curl -fsSL https://raw.githubusercontent.com/affaan-m/everything-claude-code/main/install.sh | bash
   ```
2. **Инициализация в проекте:**
   Откройте Claude Code и введите команду:
   ```text
   /plugin install everything-claude-code@everything-claude-code
   ```

## Использование ролей (Примеры запуска)

Вместо того чтобы сразу просить агента писать код, вы переключаете его в режим узкого специалиста. На этом этапе закладываются принципы Domain-Driven Design (DDD).

### Роль "Планировщик" (Planner)
Задействуется на начальном этапе для разбивки идей на эпики и подзадачи.

*Пример команды в чате:*
> "Действуй как Планировщик (Planner). Разбей идею создания маркетплейса на эпики и подзадачи. Сохрани план в `plan.md`."

### Роль "Архитектор" (Architect)
Отвечает за архитектуру системы, микросервисы и API.

*Пример команды в чате:*
> "Переключись на роль Архитектора. Изучи `plan.md` и спроектируй схему базы данных PostgreSQL и архитектуру микросервисов. Запиши результат в `architecture.md`."

### Роль "Разработчик" (Developer) со скиллами Superpowers
Отвечает за написание функционала, соблюдая жесткую инженерную дисциплину фреймворка Superpowers. Вместо бесконтрольной кодогенерации работает по процессу: мозговой штурм -> TDD (написание падающих тестов) -> реализация до "зеленых" тестов.

*Пример команды в чате:*
> "Действуй как Разработчик (Developer), используя воркфлоу Superpowers. Реализуй компонент корзины: сначала напиши unit-тесты (TDD), покажи их мне, а затем напиши код реализации, пока тесты не пройдут."

### Роль "Ревьюер" (Reviewer) со скиллами Superpowers
Выступает в качестве строгого QA-инженера и Senior-разработчика. Проводит двухэтапную проверку качества (в рамках подходов Superpowers и Subagent-Driven Development), валидирует изменения на соответствие `CLAUDE.md`, спецификациям и архитектуре.

*Пример команды в чате:*
> "Переключись на роль Строгого Ревьюера (Reviewer). Проверь последние изменения в `src/api.ts` на соответствие контрактам и спецификациям. Найди уязвимости, проблемы с оптимизацией и выведи список замечаний для исправления."

### Роль "Поиск багов" (Bug Hunter) со скиллами Superpowers
Основан на 4-фазном процессе поиска первопричин (Root Cause Analysis) из Superpowers. Систематически трассирует ошибку от симптомов до корня проблемы, исключая исправление методом "тыка". Предотвращает регрессию.

*Пример команды в чате:*
> "Действуй как Bug Hunter (Поиск багов). В компоненте оплаты падает ошибка. Используй системную отладку Superpowers: найди первопричину, предложи исправление и напиши тест, предотвращающий повторное появление бага."

Используя разные роли, вы избегаете ситуации, когда ИИ начинает писать код до того, как продумана архитектура, и получаете более качественный результат проектирования.

## Практический пример: Разворачиваем Vue.js проект с "мозгами"

Допустим, вы хотите создать новый проект на Vue.js и сразу подключить к нему интеллект (навыки и роли) из `everything-claude-code`.

**Шаг 1. Инициализация базы (Фундамент)**
Сначала вы используете утилиту-шаблонизатор, чтобы заложить базовые правила и настроить серверы MCP.
```bash
# 1. Создаем папку и переходим в нее
mkdir my-vue-app && cd my-vue-app

# 2. Генерируем базовые контракты (CLAUDE.md, настройки)
npx claude-code-templates
```
*(В интерактивном меню выберите профиль Frontend/Vue).*

**Шаг 2. Развертывание фреймворка Vue**
Откройте Claude Code (`claude`) в этой папке и попросите его сгенерировать шаблон кода:
> "Разверни новый проект Vue.js с использованием Vite и TypeScript в текущей директории."

**Шаг 3. Добавление "мозгов" (Роли и Навыки)**
Теперь, когда базовый код и контракты есть, мы "нанимаем" команду из `everything-claude-code`:
```text
# Команда внутри Claude Code для подключения ролей
/plugin install everything-claude-code@everything-claude-code
```

**Шаг 4. Работа по ролям (Spec-Driven Development)**
Теперь вы не просите ИИ просто "сделай форму логина". Вы оркестрируете работу как Tech Lead:

1. **Планировщик:**
   > "Действуй как Планировщик. Изучи мой Vue проект и создай `plan.md` с декомпозицией компонентов для системы аутентификации."
2. **Разработчик:**
   > "Переключись на роль Разработчика. Следуя `plan.md`, напиши unit-тесты (TDD) для компонента `LoginForm.vue`. Убедись, что они падают. Только после этого реализуй сам компонент."
3. **Ревьюер:**
   > "Переключись на роль Ревьюера. Проверь созданный `LoginForm.vue` на соответствие стилю Vue 3 Composition API. Убедись, что пройдены все обязательные гейты из `CLAUDE.md` (линтер, `vue-tsc`, тесты)."

Таким образом, `claude-code-templates` заливает инфраструктурный фундамент (настройки среды), а `everything-claude-code` дает агенту дисциплину и разделение труда при написании самого кода проекта.
